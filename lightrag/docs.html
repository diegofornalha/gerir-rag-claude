LightRAG - Documentação

LightRAG é um sistema leve de RAG (Retrieval Augmented Generation) para uso com Claude via MCP. Esta implementação oferece uma maneira simples de armazenar e recuperar informações contextuais para melhorar as respostas do Claude.

Instalação

Para instalar o LightRAG, execute o script de instalação:

cd ~/.claude/lightrag
./setup_lightrag.sh

Isso irá:

  • Verificar e instalar as dependências necessárias
  • Configurar os diretórios e permissões
  • Criar links simbólicos necessários
  • Configurar a integração MCP
  • Configurar o serviço para iniciar automaticamente
  • Iniciar o servidor

Uso Básico

Usando no código Python ou dentro do Claude

from claude import MCP

# Conectar ao serviço
lightrag = MCP.connect_to_service('lightrag')

# Adicionar informações à base de conhecimento
lightrag.rag_insert_text(text="LightRAG é um sistema de RAG leve para Claude.")

# Consultar informações
resultado = lightrag.rag_query(query="O que é LightRAG?")
print(resultado)

Usando o demo interativo

cd ~/.claude/lightrag
python3 demo.py

Isso iniciará uma interface interativa para testar o LightRAG.

Testando via cURL

# Consulta
curl -X POST -H "Content-Type: application/json" \
     -d '{"query":"O que é LightRAG?"}' \
     http://127.0.0.1:5000/query

# Inserção
curl -X POST -H "Content-Type: application/json" \
     -d '{"text":"LightRAG é um sistema de RAG leve."}' \
     http://127.0.0.1:5000/insert

# Status
curl http://127.0.0.1:5000/status

API do LightRAG

Endpoints HTTP

Endpoint Método Parâmetros Descrição
/query POST {"query": "texto da consulta", "max_results": 5} Consulta a base de conhecimento
/insert POST {"text": "texto a inserir", "source": "origem"} Insere texto na base de conhecimento
/status GET Nenhum Retorna status do servidor
/clear POST {"confirm": true} Limpa a base de conhecimento

Funções MCP

Função Parâmetros Descrição
rag_query
  • query: texto da consulta
  • mode: modo de consulta (padrão: "hybrid")
  • max_results: número máximo de resultados (padrão: 5)
 Consulta a base de conhecimento
rag_insert_text
  • text: texto a inserir
  • source: origem do texto (padrão: "mcp")
 Insere texto na base de conhecimento
rag_status Nenhum Retorna status do servidor
rag_clear
  • confirm: confirmação (padrão: True)
 Limpa a base de conhecimento

Estrutura de Arquivos

  • micro_lightrag.py - Servidor Flask principal
  • lightrag_mcp.py - Cliente MCP para integração com Claude
  • start_lightrag.sh - Script para iniciar o servidor
  • setup_lightrag.sh - Script de instalação completa
  • demo.py - Aplicativo de demonstração interativo
  • lightrag_db.json - Banco de dados (JSON)
  • com.user.lightrag.plist - Configuração para inicialização automática
  • logs/ - Diretório de logs
Nota: O LightRAG utiliza um banco de dados JSON simples armazenado em lightrag_db.json. Para cargas maiores, considere implementar uma solução de banco de dados mais robusta.

Solução de Problemas

O servidor não inicia

Verifique os logs em ~/.claude/lightrag/logs/ para identificar o problema. Certifique-se de que a porta 5000 não está sendo usada por outro aplicativo.

Erro de conexão recusada

Verifique se o servidor está em execução. Você pode reiniciá-lo com:

~/.claude/start_lightrag.sh

Integração MCP não funciona

Certifique-se de que o MCP está configurado corretamente. Você pode testar diretamente usando:

python3 -c "from claude import MCP; lightrag = MCP.connect_to_service('lightrag'); print(lightrag.rag_status())"
Aviso: O LightRAG é um sistema simples e não foi projetado para ambientes de produção com alta carga. Use-o para propósitos de desenvolvimento e teste.
